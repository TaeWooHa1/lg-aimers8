❌ 문제 2: PerfNorm 측정 방식이 다르다  
python
# evaluate.py (line 199-201)
result.perf_norm = baseline.perplexity / result.perplexity
대회	evaluate.py
성능 지표	비공개 벤치마크 정확도 (accuracy)	Perplexity
데이터	비공개 벤치셋	고정 텍스트 3개
왜 문제인가?

Perplexity ≠ 정확도. Perplexity가 낮다고 벤치마크 정확도가 높다는 보장이 없습니다. 특히 양자화 모델은 perplexity 변화가 작아도 특정 태스크 정확도가 크게 떨어질 수 있습니다.
Perplexity 측정에 텍스트 3줄만 사용합니다 (line 164-168). 통계적으로 신뢰할 수 없습니다.

⚠️ 문제 5: PerfNorm > 1.0이 될 수 있다
python
result.perf_norm = baseline.perplexity / result.perplexity  # line 201
양자화 모델의 perplexity가 우연히 기본 모델보다 낮으면 PerfNorm > 1.0이 됩니다. 실제 대회에서도 가능은 하지만, 3줄의 텍스트로 측정하면 노이즈에 의한 가짜 결과일 가능성이 높습니다.
