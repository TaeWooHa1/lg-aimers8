# FP8 양자화 개념 정리

## 1. FP8이란?

FP8(8-bit Floating Point)은 **8비트 부동소수점** 형식으로, 기존 FP16(16비트)의 절반 크기로 숫자를 표현합니다.

```
FP16: [1 sign][5 exponent][10 mantissa] = 16bit
FP8:  [1 sign][4 exponent][ 3 mantissa] = 8bit  (E4M3)
FP8:  [1 sign][5 exponent][ 2 mantissa] = 8bit  (E5M2)
```

### E4M3 vs E5M2

| 형식 | 지수(E) | 가수(M) | 범위 | 정밀도 | 용도 |
|------|---------|---------|------|--------|------|
| **E4M3** | 4비트 | 3비트 | ±448 | 높음 | **가중치 & 순전파 활성화** |
| **E5M2** | 5비트 | 2비트 | ±57344 | 낮음 | 역전파 그래디언트 |

> 추론(inference)에서는 **E4M3**만 사용됩니다.

### INT4 vs FP8 차이

```
INT4:  정수 4비트 → 0~15 (16개 값) → 디퀀타이즈 후 FP16으로 연산
FP8:   부동소수점 8비트 → ~256개 값 → FP8 Tensor Core에서 직접 연산!
```

**핵심 차이**: INT4는 연산 시 FP16으로 변환 필요, FP8은 **하드웨어에서 직접 연산** 가능

## 2. Weight(가중치)와 Activation(활성화)

### Weight (가중치)

모델이 학습을 통해 얻은 **고정된 파라미터**입니다.

```
입력 → [가중치 W 곱하기] → 출력

y = W × x + b
    ↑ 이것이 Weight
```

- 모델 파일(safetensors)에 저장됨
- 추론 시 **변하지 않음** (고정값)
- 모델 크기 = 가중치의 총 크기

### Activation (활성화)

추론 중 **각 레이어의 입출력 값**으로, 입력 데이터에 따라 매번 달라집니다.

```
"안녕하세요" → [Layer 1] → activation₁ → [Layer 2] → activation₂ → ... → 출력
                              ↑ 매번 다른 값!
```

- 메모리에만 존재 (저장 안 됨)
- 입력마다 **매번 변함** (동적)
- GPU VRAM을 차지

### 양자화 표기법

| 표기 | Weight | Activation | 설명 |
|------|--------|------------|------|
| **W4A16** | 4bit INT | 16bit FP | 가중치만 양자화, 연산은 FP16 |
| **W8A8 INT** | 8bit INT | 8bit INT | INT8 Tensor Core 활용 |
| **W8A8 FP8** | 8bit FP8 | 8bit FP8 | FP8 Tensor Core 활용 ⭐ |

## 3. 양자화 방식: Static vs Dynamic

### Static (정적) 양자화
```
[캘리브레이션 데이터로 스케일 미리 계산] → 추론 시 고정 스케일 사용

장점: 추론 시 스케일 계산 오버헤드 없음 → 약간 더 빠름
단점: 캘리브레이션 데이터 필요, 데이터 분포 변화에 취약
```

### Dynamic (동적) 양자화
```
[추론 시 매 토큰마다 스케일 실시간 계산] → 입력에 맞는 최적 스케일

장점: 캘리브레이션 불필요, 어떤 입력에도 최적 스케일 → 정확도 ↑
단점: 스케일 계산 오버헤드 (매우 작음)
```

> **추천: FP8_DYNAMIC** — 동적 활성화 양자화로 정확도가 더 높고, 속도 차이는 미미

## 4. FP8 vs INT4(W4A16) vs INT8(W8A8) 비교

| 항목 | W4A16 (현재) | W8A8 INT8 | **W8A8 FP8** |
|------|-------------|-----------|-------------|
| 모델 크기 | **~0.7GB** ⭐ | ~1.2GB | ~1.2GB |
| 정확도 | 중간 | 높음 | **매우 높음** ⭐ |
| 연산 방식 | dequant→FP16 | INT8 TC | **FP8 TC** ⭐ |
| 추론 속도 | 빠름 | 매우 빠름 | **매우 빠름** ⭐ |
| L4 지원 | ✅ (간접) | ✅ | ✅ (네이티브) |

### 왜 FP8이 INT4보다 빠를 수 있는가?

```
W4A16 추론 과정:
  메모리 → [4bit 가중치 읽기] → [FP16으로 디퀀타이즈] → [FP16 행렬곱] → 출력
                                  ↑ 변환 비용 발생!

FP8 추론 과정:
  메모리 → [FP8 가중치 읽기] → [FP8 Tensor Core 직접 연산!] → 출력
                                ↑ 변환 없이 바로 연산!
```

- W4A16: 모델은 작지만 연산은 FP16 → **memory bandwidth bound**
- FP8: 모델은 약간 크지만 FP8 Tensor Core 직접 활용 → **compute efficient**

---

## 5. L4 GPU와 FP8

L4 GPU (Ada Lovelace 아키텍처)의 Tensor Core 지원:

| 데이터 타입 | Tensor Core | TFLOPS |
|------------|-------------|--------|
| FP32 | ❌ | ~30 |
| FP16/BF16 | ✅ | ~120 |
| INT8 | ✅ | ~242 |
| **FP8** | ✅ | **~242** ⭐ |

> FP8은 INT8과 동등한 처리량을 가지면서, **부동소수점이라 정확도가 더 높음**

---

## 6. 이 대회에서 FP8이 유리한 이유

```
Score = 0.5 × PerfNorm + 0.5 × SpeedNorm
```

| 전략 | PerfNorm (정확도) | SpeedNorm (속도) | 예상 Score |
|------|-------------------|------------------|-----------|
| W4A16 기본 | ~0.92 | ~0.51 | ~0.71 |
| W4A16 최적화 | ~1.03 | ~0.51 | ~0.77 |
| **FP8** | **~0.98~1.0** | **~0.65~0.75** | **~0.83~0.87** ⭐ |

- **PerfNorm**: FP8은 거의 무손실 → base 모델 대비 ~98~100% 유지
- **SpeedNorm**: FP8 Tensor Core → 토큰당 시간 대폭 감소 → SpeedNorm ↑↑↑

> FP8은 **정확도와 속도 모두에서 이점**이 있어, 50:50 평가에서 최적의 선택
