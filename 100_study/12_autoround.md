|---------|---------|------|

## ⚖️ `group_size` vs `AUTOROUND_BATCH_SIZE` 차이점

| 구분 | **`group_size`** (양자화 그룹 크기) | **`batch_size`** (학습 배치 크기) |
| :-- | :-- | :-- |
| **의미** | 모델 가중치(Parameter)를 **몇 개씩 묶어서 양자화할지** 결정 | 양자화 최적화(Tuning) 시 **한 번에 몇 개의 데이터를 사용할지** 결정 |
| **영향 시점** | **추론 시 (Inference)** | **양자화 과정 중 (Training/Quantization)** |
| **역할** | 모델의 **정확도**와 **추론 속도** 결정 | 양자화 **실행 속도**와 **GPU 메모리 사용량** 결정 |
| **정확도 영향** | 큼 (작을수록 정확) | 거의 없음 (너무 작으면 학습 불안정) |
| **추론 속도 영향** | 큼 (클수록 빠름, `-1`일 때 가장 빠름) | **없음** (최종 모델 성능과 무관) |
| **메모리 영향** | 모델 크기(VRAM)에 영향 | 양자화 중 사용량(VRAM)에 영향 |
| **추천 값** | `128` (균형), `-1` (속도), `32` (정확도) | `8` (T4 기준), `32` (VRAM 충분 시) |

### 1. `group_size` (Weight Grouping) - **미설정 (기본값 128)**
- 가중치를 그룹핑하여 스케일(Scale)과 제로 포인트(Zero Point)를 공유하는 단위.
- **128 (기본)**: 128개 가중치마다 파라미터 1개 추가. (정확도/속도 균형)
- **-1 (Channel-wise)**: 채널 전체를 1개 그룹으로. (가장 빠름, 정확도 낮음)
- **32**: 32개마다 파라미터 추가. (가장 정확, 느리고 메모리 큼)

### 2. `batch_size` (Calibration Batch) - **설정값 8 (수정됨: 32)**
- AutoRound가 200번(`iters`) 반복 학습할 때, 한 번(1 step)에 모델에 흘려보내는 데이터 개수.
- **8**: T4 GPU(16GB)에서 OOM 없이 안정적.
- **32**: A100 등 큰 GPU에서 학습 속도 향상. (결과물 품질엔 차이 없음)

## ❗ 설정되지 않은 파라미터 (정확도·추론속도에 영향)

1. **`group_size`** (기본값: 128)
   - W4A16 스킴 내부 기본값 사용 중
   - 작을수록(64, 32) 정확도 ↑ 추론속도 ↓ / 클수록(256) 정확도 ↓ 추론속도 ↑
   - 이걸 `scheme` 대신 직접 `config_groups`로 지정하면 세밀 제어 가능

2. **`symmetric`** (기본값: True)
   - W4A16 기본값은 대칭(symmetric) 양자화
   - `False`(비대칭)로 바꾸면 정확도가 약간 올라갈 수 있으나 추론 속도 영향

3. **`enable_torch_compile`** (기본값: True)
   - `torch.compile`로 양자화 튜닝 루프를 가속
   - **양자화 실행 속도**에 영향 (최종 추론 속도와는 무관)
   - 환경에 따라 컴파일 오류 시 `False`로 설정 필요

4. **`MAX_SEQUENCE_LENGTH = 512`** ⚠️
   - 현재 512인데, llm-compressor 공식 권장 기본값은 **2048**
   - 더 긴 시퀀스로 캘리브레이션하면 **정확도가 향상**될 수 있음
   - Kaggle 메모리가 허용하면 `1024` 또는 `2048`로 올리는 것 추천

5. **llm-compressor 추천 설정 비교**
   | 모드 | iters | seqlen | nsamples | batch_size | 정확도 |
   |------|-------|--------|----------|------------|--------|
   | **default** | 200 | **2048** | 128 | 8 | 🎯🎯 Good |
   | **best** | 1000 | **2048** | 512 | 8 | 🏆 Best |
   | **light** | 50 | **2048** | 128 | 8 | 🎯🎯 (약간 드롭) |
   | **현재 코드** | 200 | **512** | **1024** | **32** | ❓ |

   > 현재 코드는 `default` 모드에 가깝지만 `seqlen`만 512로 낮은 상태입니다. **`MAX_SEQUENCE_LENGTH`를 1024~2048로 올리는 것이 정확도 개선에 가장 큰 영향**을 줄 수 있습니다. 메모리가 부족하면 `batch_size`를 4로 낮추는 것도 방법입니다.
