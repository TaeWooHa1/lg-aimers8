## 🔴 직접 영향 변수 (GPTQModifier)

### 1. `scheme` - 양자화 스키마

| 항목 | 설명 |
|------|------|
| **역할** | 가중치(W)와 활성화(A)의 비트 수 결정 |
| **기본값** | `"W4A16"` |
| **옵션** | `"W4A16"`, `"W8A16"`, `"W8A8"` |

**사용 이유**: 모델 크기와 정확도의 균형점 결정

| 값 | 의미 | 압축률 | 정확도 |
|----|------|--------|--------|
| `W4A16` | 4비트 가중치, 16비트 활성화 | ~3.7x | 보통 |
| `W8A16` | 8비트 가중치, 16비트 활성화 | ~1.8x | 높음 |
| `W8A8` | 8비트 가중치, 8비트 활성화 | ~1.8x | 낮음 |

**권장**: `W4A16` (크기-성능 최적 균형)

---

### 2. `targets` - 양자화 대상 레이어

| 항목 | 설명 |
|------|------|
| **역할** | 어떤 레이어 타입을 양자화할지 지정 |
| **기본값** | `["Linear"]` |
| **옵션** | `["Linear"]`, `["Embedding", "Linear"]` 등 |

**사용 이유**: Linear 레이어가 파라미터의 대부분을 차지하므로 주 타겟

```python
targets = ["Linear"]  # 가장 일반적인 설정
```

---

### 3. `ignore` - 양자화 제외 레이어

| 항목 | 설명 |
|------|------|
| **역할** | 특정 레이어를 양자화에서 제외 |
| **기본값** | `[]` |
| **권장값** | `["embed_tokens", "lm_head"]` |

**사용 이유**: 
- `embed_tokens`: 입력 토큰 → 벡터 변환 (정밀도 필요)
- `lm_head`: 최종 출력층 (예측 정확도에 직접 영향)

| 설정 | 효과 |
|------|------|
| `[]` | 모든 레이어 양자화 (최대 압축) |
| `["lm_head"]` | 출력층 보호 |
| `["embed_tokens", "lm_head"]` | ⭐ 권장 (입출력 보호) |

---

### 4. `actorder` - 활성화 순서 정렬

| 항목 | 설명 |
|------|------|
| **역할** | 가중치 컬럼의 양자화 순서 결정 |
| **기본값** | `"static"` |
| **옵션** | `"static"`, `"group"`, `"weight"`, `"dynamic"` |

**사용 이유**: 중요한 가중치를 먼저/나중에 처리하여 오차 최소화

| 값 | 설명 | 정확도 | 속도 |
|----|------|--------|------|
| `"static"` | 고정 순서 | 기준 | 빠름 |
| `"group"` | 그룹 단위 정렬 | ↑ | 보통 |
| `"weight"` | 가중치 크기 순 | ↑↑ | 느림 |
| `"dynamic"` | 동적 결정 | ↑↑↑ | 매우 느림 |

**권장**: 
- 정확도 우선: `"weight"`
- 속도 우선: `"static"`

---

### 5. `dampening_frac` - Hessian 감쇠 비율

| 항목 | 설명 |
|------|------|
| **역할** | Hessian 행렬 대각선에 감쇠 적용 |
| **기본값** | `0.01` |
| **범위** | `0.001` ~ `0.1` |

**사용 이유**: 
- Hessian 역행렬 계산의 수치적 안정성 확보
- 양자화 실패/NaN 방지

| 값 | 효과 |
|----|------|
| `0.001` | 감쇠 약함 (불안정 가능) |
| `0.01` | ⭐ 기본값 |
| `0.1` | 감쇠 강함 (안정, 정확도 약간↓) |

**팁**: 양자화 중 NaN/오류 발생 시 값을 높이기

---

### 6. `group_size` - 그룹 크기

| 항목 | 설명 |
|------|------|
| **역할** | 함께 양자화되는 가중치 수 |
| **기본값** | `128` |
| **옵션** | `32`, `64`, `128`, `256` |

**사용 이유**: 
- 작은 그룹 = 더 세밀한 양자화 = 높은 정확도
- 큰 그룹 = 더 빠른 추론 = 작은 모델

| 값 | 정확도 | 모델 크기 | 추론 속도 |
|----|--------|----------|----------|
| `32` | ↑↑↑ | ↑ | ↓ |
| `64` | ↑↑ | 약간↑ | ↓ |
| `128` | 기준 | 기준 | 기준 |
| `256` | ↓ | ↓ | ↑ |

**권장**: `128` (균형) 또는 `64` (정확도 우선)

---

### 7. `block_size` - 블록 크기 ✨ NEW

| 항목 | 설명 |
|------|------|
| **역할** | 한 번에 압축할 가중치 컬럼 수 |
| **기본값** | `128` |
| **옵션** | `64`, `128`, `256` |

**사용 이유**: 
- GPTQ 알고리즘 내부에서 가중치 행렬을 블록 단위로 처리
- 작은 블록 = 더 정밀한 오차 보정 = 높은 정확도
- 큰 블록 = 더 빠른 양자화 시간

| 값 | 정확도 | 양자화 시간 |
|----|--------|------------|
| `64` | ↑ | 느림 |
| `128` | 기준 | 기준 |
| `256` | ↓ | 빠름 |

```python
GPTQModifier(..., block_size=128)
```

---

### 8. `symmetric` - 대칭 양자화 ✨ NEW

| 항목 | 설명 |
|------|------|
| **역할** | 양자화 범위를 0 기준 대칭으로 설정할지 결정 |
| **기본값** | `False` (비대칭) |
| **옵션** | `True`, `False` |

**사용 이유**: 
- **대칭 (True)**: 범위가 `-max ~ +max`, 속도 빠름
- **비대칭 (False)**: 범위가 `min ~ max`, 정확도 높음 (AWQ 스타일)

| 값 | 정확도 | 추론 속도 |
|----|--------|----------|
| `True` | ↓ | ↑ |
| `False` | ↑ | ↓ |

```python
# scheme 내부에서 설정 또는
GPTQModifier(..., symmetric=False)
```

**권장**: `False` (정확도 우선)

---

### 9. `sequential_targets` - 순차 처리 대상 ✨ NEW

| 항목 | 설명 |
|------|------|
| **역할** | 순차적으로 압축할 레이어 블록 지정 |
| **기본값** | `None` (자동 추론) |
| **옵션** | `["LlamaDecoderLayer"]`, `["ExaoneDecoderLayer"]` 등 |

**사용 이유**: 
- 레이어 단위로 순차 처리 시 메모리 사용량 감소
- 이전 레이어의 양자화 오차를 다음 레이어 입력에 반영
- 오차 누적 방지로 정확도 향상

```python
GPTQModifier(..., sequential_targets=["ExaoneDecoderLayer"])
```

**효과**:
| 설정 | 메모리 | 정확도 | 시간 |
|------|--------|--------|------|
| `None` | 높음 | 기준 | 빠름 |
| 레이어 지정 | ↓↓↓ | ↑ | 느림 |

---

### 10. `config_groups` - 레이어별 설정 그룹 ✨ NEW

| 항목 | 설명 |
|------|------|
| **역할** | 서로 다른 레이어에 다른 양자화 설정 적용 |
| **기본값** | `None` (단일 설정) |
| **고급 기능** | ⭐⭐⭐⭐⭐ |

**사용 이유**: 
- 어텐션 레이어는 8비트, MLP는 4비트 등 세분화 가능
- 중요 레이어 보호하면서 압축률 최적화

```python
config_groups = {
    "attention_weights": {
        "targets": ["q_proj", "k_proj", "v_proj", "o_proj"],
        "scheme": "W8A16",  # 어텐션은 8비트
    },
    "mlp_weights": {
        "targets": ["gate_proj", "up_proj", "down_proj"],
        "scheme": "W4A16",  # MLP는 4비트
    }
}
```

**효과**: 정밀한 레이어별 최적화 가능

---

### 11. `offload_hessians` - Hessian 오프로드 ✨ NEW

| 항목 | 설명 |
|------|------|
| **역할** | Hessian 행렬을 CPU로 오프로드하여 GPU 메모리 절약 |
| **기본값** | `False` |
| **옵션** | `True`, `False` |

**사용 이유**: 
- 대형 모델 양자화 시 GPU 메모리 부족 해결
- 속도는 느려지지만 OOM 방지

| 값 | GPU 메모리 | 양자화 시간 |
|----|-----------|------------|
| `False` | 높음 | 빠름 |
| `True` | ↓↓ | 느림 |

```python
GPTQModifier(..., offload_hessians=True)
```

## 🟡 간접 영향 변수 (oneshot)

### 12. `num_calibration_samples` - 캘리브레이션 샘플 수

| 항목 | 설명 |
|------|------|
| **역할** | Hessian 행렬 추정에 사용되는 샘플 수 |
| **기본값** | `256` |
| **권장** | `256` ~ `1024` |

**사용 이유**: 
- 더 많은 샘플 = 더 정확한 Hessian 추정 = 낮은 양자화 오차
- 다양한 입력 패턴 커버

| 값 | 정확도 | 양자화 시간 | 메모리 |
|----|--------|------------|--------|
| `128` | ↓ | 빠름 | 낮음 |
| `256` | 기준 | 기준 | 기준 |
| `512` | ↑ | 2x | 높음 |
| `1024` | ↑↑ | 4x | 매우 높음 |

---

### 13. `max_seq_length` - 최대 시퀀스 길이

| 항목 | 설명 |
|------|------|
| **역할** | 캘리브레이션 시 처리할 최대 토큰 수 |
| **기본값** | `512` |
| **권장** | `512` ~ `2048` |

**사용 이유**: 
- 긴 시퀀스 = 다양한 위치의 활성화 패턴 학습
- RoPE 등 위치 임베딩 영향 반영

| 값 | 정확도 | 메모리 사용 |
|----|--------|-----------|
| `256` | ↓ | 낮음 |
| `512` | 기준 | 기준 |
| `1024` | ↑ | 2x |
| `2048` | ↑↑ | 4x |

---

### 14. `dataset` - 캘리브레이션 데이터셋

| 항목 | 설명 |
|------|------|
| **역할** | 양자화 시 활성화 패턴 추출에 사용 |
| **권장** | 실제 사용 도메인과 유사한 데이터 |

| 데이터셋 | 특징 |

## 🔄 변수에 영향을 주는 상위 요인 (Meta-Variables)

> 양자화 파라미터 선택은 단독 결정이 아닙니다. 다음 상위 요인들이 최적 값에 영향을 줍니다.

### 📊 계층적 영향 관계

```
🌍 외부 환경
├── 하드웨어 제약
│   ├── GPU 메모리 → num_calibration_samples, max_seq_length, offload_hessians
│   ├── GPU 종류 → device_map, torch_dtype
│   └── 연산 시간 예산 → actorder, block_size
│
├── 모델 특성
│   ├── 모델 크기 → scheme, group_size
│   ├── 아키텍처 → ignore, sequential_targets, trust_remote_code
│   └── 레이어 구조 → config_groups, targets
│
├── 데이터 특성
│   ├── 도메인 유사성 → dataset 선택
│   └── 데이터 다양성 → num_calibration_samples
│
└── 목표/제약
    ├── 정확도 목표 → actorder, dampening_frac, symmetric
    ├── 속도 목표 → scheme, group_size
    └── 제출 형식 → save_compressed
```

---

### 🎯 변수별 영향 요인 매핑

| 양자화 변수 | 영향 받는 상위 요인들 |
|------------|---------------------|
| `num_calibration_samples` | GPU 메모리, 양자화 시간 예산, 데이터셋 다양성 |
| `max_seq_length` | GPU 메모리, 실제 사용 컨텍스트 길이, 모델 특성 |
| `scheme` | 목표 압축률, 허용 정확도 손실, 추론 하드웨어 |
| `actorder` | 시간 예산, 정확도 요구사항 |
| `group_size` | 모델 크기, 메모리 제약, vLLM 호환성 |
| `ignore` | 모델 아키텍처, 레이어별 민감도 분석 |
| `dampening_frac` | 수치 안정성, 모델 가중치 분포 |
| `dataset` | 실제 사용 도메인, 모델 학습 데이터 |
| `config_groups` | 레이어별 중요도, 혼합 정밀도 전략 |
| `sequential_targets` | GPU 메모리, 모델 레이어 구조 |
| `symmetric` | 정확도 목표, 추론 속도 요구사항 |
| `block_size` | 양자화 시간 예산, 정밀도 요구사항 |
