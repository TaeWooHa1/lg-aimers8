1. 개인 또는 팀 참여 규칙

개인 또는 팀을 이루어 참여할 수 있습니다.
개인 참가 방법: 팀 신청 없이, 자유롭게 제출 탭에서 제출 가능
팀 참가 방법: 팀 탭에서 가능, 상세 내용은 팀 탭에서 팀 병합 정책 확인
팀 구성 방법: 팀 페이지에서 팀 구성 안내 확인
팀 최대 인원: 5명
동일인이 개인 또는 복수 팀에 중복하여 등록 불가
  

2. 대회 규칙

	1) 대회 지정 기본 모델

LGAI-EXAONE/EXAONE-4.0-1.2B [허깅페이스 링크]
	※ 본 해커톤에 제출하는 모델은 기본 모델인 EXAONE-4.0-1.2B로부터 참가자가 직접 파생한 경량화 모델로 한정됩니다.



	2) vLLM 커스컴 수정 불가

온라인 해커톤(Phase2)에서는 vLLM 추론 엔진을 고정된 환경으로 사용합니다.
참가자는 vLLM 소스 코드, CUDA 커널, 내부 로직을 수정할 수 없습니다.


	3) 외부 데이터 사용 가능

참가자가 모델 학습 또는 튜닝이 필요한 경우 외부 데이터 사용이 가능합니다.
단, 최소한 비상업적 이용이 허용된 라이선스(CC BY-NC, CC0 등)로 배포된 외부 데이터만 사용할 수 있습니다. 해당 조건을 충족하지 않는 외부 데이터는 사용이 불가능합니다.


	4) 모델 경량화 및 성능 유지·향상을 위한 추가 학습/튜닝

참가자는 제공된 기본 모델인 EXAONE-4.0-1.2B 모델을 대상으로, 모델 경량화뿐만 아니라 기본 모델 성능의 유지 또는 향상을 목적으로 추가 학습 또는 파라미터 튜닝을 수행할 수 있습니다.
모델 경량화를 위한 기법에는 제한이 없으며, 적용 방식과 무관하게 최종 결과물이 허깅페이스(Hugging Face) 표준 형식의 모델 가중치 및 설정 파일로 제출 가능한 경우에 한해 평가 대상이 됩니다.
제출된 모델은 운영진이 제공하는 동일한 vLLM 추론 환경에서 평가됩니다.

온라인 해커톤(Phase2) 종료 후 오프라인 해커톤(Phase3) 진출 희망 팀은 아래의 양식에 맞추어 코드를 dacon@dacon.io로 기한 내에 제출
o 코드 파일 확장자: .py, .ipynb

	[제출 파일 목록]

			o (필수) Public Score 재현용 학습/경량화 코드 (= 리더보드에 제출한 model 폴더를 동일하게 생성할 수 있는 코드)

5. 유의 사항

1일 최대 제출 횟수: 3회
사용 가능 언어: Python
대회 직후 공개되는 Public 랭킹은 예선 최종 순위가 아니며 코드 검증 후 본선 진출자가 결정됩니다.
코드 제출 기능을 악용한 의도적인 평가 데이터셋 유출 등을 포함한 규칙 위반 사항이 발견되는 경우 실격에 해당합니다.
LLM 기반 모델의 비결정적 실행 특성 및 평가 서버 환경(시스템 부하, 스케줄링 등)에 따라, 동일한 제출물(제출 모델)이라 하더라도 리더보드 성능은 제출 시점마다 일부 차이가 발생할 수 있습니다.
이에 따라 제출 시 평가 서버에서 실제로 측정·기록된 리더보드 점수만을 공식 평가 결과로 인정합니다.
